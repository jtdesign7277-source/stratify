  <name>Task 2: Create useBetHistory hook and computeStats function</name>
  <files>src/hooks/useBetHistory.js</files>
  <action>
Create src/hooks/useBetHistory.js with two named exports:

1. **useBetHistory()** hook:
   - Uses cancelled-flag pattern from SportsBankroll.jsx (see research Pattern 1)
   - Calls supabase.auth.getSession() to get user_id
   - If no session, set loading=false and return early
   - Queries: supabase.from('paper_sports_bets').select('*').eq('user_id', session.user.id).order('created_at', { ascending: false })
   - Returns { bets, loading, error }
   - bets is an array (default []), loading starts true, error is null or string
   - Cleanup: cancelled = true on unmount

2. **computeStats(bets)** pure function:
   - Accepts bets array (the full array or a filtered subset — supports STAT-02 for Phase 3)
   - Computes exactly these four values:
     - totalWagered: sum of bet_amount on ALL bets (including pending)
     - totalWon: sum of actual_payout on bets where status === 'won'
     - netPnl: (sum of actual_payout on won bets) - (sum of bet_amount on won bets) - (sum of bet_amount on lost bets). Pending bets excluded from realized P&L.
     - winRate: won.length / (won.length + lost.length) * 100. If no settled bets, return 0. Pending excluded from denominator (per STATE.md convention).
   - Returns { totalWagered, totalWon, netPnl, winRate }
   - All numeric fields coerced with Number() to handle string values from Supabase

Import supabase from '../lib/supabaseClient' (relative path).
Do NOT import anything from sportsUtils.js in this file (no calcPayout needed — stats use actual_payout from DB).

Add a comment block at top documenting the verified column names from Task 1.
  </action>
